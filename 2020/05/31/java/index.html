二叉树

特点：

1、所有节点最多拥有两个子节点，即度不大于2
2、左子树的键值小于根的键值，右子树的键值大于根的键值。

红黑树

特点：

1、通过旋转重新达到自平衡，依然保留二叉树特点。

B+树

1、mysql索引中使用到b+树

  • 非叶子节点不存储data，只存储索引（冗余），可以放更多的索引。
  • 叶子节点包含所有索引字段。
  • 叶子节点用指针链接，提高区间访问性能。

jvm虚拟机

  • 堆：new出来的对象放在堆里
  • z栈：每个线程都会去开辟一个栈空间，main也是。
  • 方法区：常量、静态变量、类元数据（.class文件）

堆

  • 堆默认大小600M

  • Eden存放的是新 new 出来的对象，当放满是会触发 gc。

    清理原则：对象没有被引用类型引用。

    GC会对Eden和Survivor区都进行垃圾回收，对象年代经过了15次垃圾回收才会进入老年代区。

多线程内存模型

  • 每开一个线程jvm都会将主内存的共享变量copy一份到当前线程内存。

spring security

spring security 是一个认证的框架，可以自定义 UserDetails、UserDetailsService、passwordEncoder、配置类：WebSecurityConfigurerAdapter

过滤器列表：

在类 FilterComparator 中定义。

所有过滤器都继承类 GenericFilterBean。

spring security oauth2

  1. oauth2 是一种协议，而 spring security oauth2 是实现了这种协议的框架

  2. 此框架使用场景为：

用户 平台 第三方应用；当第三方应用需要对接平台时，平台可以提供已经实现的授权并登陆页面给应用，平台为应用分配一个appid和appsecret，平台会给应用返回一个code，应用拿code取获取token，再去获取平台资源。

  1. 可以自定义 UserDetails、UserDetailsService、passwordEncoder，配置类：WebSecurityConfigurerAdapter
  2. 可以自定义 ClientDetails、ClientDetailsService、PasswordEncoder，配置类：AuthorizationServerConfigurerAdapter

cookie 和 session

cookie 和 session是服务器的一种会话技术

  • cookie

    Cookie实际上是一小段的文本信息。客户端请求服务器，如果服务器需要记录该用户状态，就使用response向客户端浏览器颁发一个Cookie。客户端会把Cookie保存起来。

    当浏览器再请求该网站时，浏览器把请求的网址连同该Cookie一同提交给服务器。服务器检查该Cookie，以此来辨认用户状态。服务器还可以根据需要修改Cookie的内容。

    1、会话Cookie和持久Cookie

    若不设置过期时间，则表示这个cookie的生命期为浏览器会话期间，关闭浏览器窗口，cookie就消失。这种生命期为浏览器会话期的cookie被称为会话cookie。会话cookie一般不存储在硬盘上而是保存在内存里，当然这种行为并不是规范规定的。

    若设置了过期时间，浏览器就会把cookie保存到硬盘上，关闭后再次打开浏览器，这些cookie仍然有效直到超过设定的过期时间。存储在硬盘上的cookie可以在浏览器的不同进程间共享。这种称为持久Cookie。

    2、Cookie具有不可跨域名性

    就是说，浏览器访问百度不会带上谷歌的cookie;

  • session

    Session是另一种记录客户状态的机制，不同的是Cookie保存在客户端浏览器中，而Session保存在服务器上。客户端浏览器访问服务器的时候，服务器把客户端信息以某种形式记录

    在服务器上。这就是Session。客户端浏览器再次访问时只需要从该Session中查找该客户的状态就可以了。

    每个用户访问服务器都会建立一个session，那服务器是怎么标识用户的唯一身份呢？事实上，用户与服务器建立连接的同时，服务器会自动为其分配一个SessionId。
